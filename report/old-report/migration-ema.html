5. Navigation Plan

▼
Unified new framework for all pages (Decision D3) — only one header and one footer variant needed.

5.1 Header

ComponentStructure
HeaderWF logo (left) + Sign On link + hamburger MENU button (right). Sign On uses LOB-specific URL param (LOB=CONS, LOB=BIZ). LOB determined by page path prefix.

5.2 Footer

ComponentStructure
FooterLegal nav links + social media icons (Facebook, LinkedIn, Instagram, Pinterest, YouTube, Twitter) + investment disclaimer + copyright

5.3 Other Navigation

ComponentApproach
Product nav barDefault content — unordered list in first section, styled via section-metadata as horizontal link bar. Not a block.
LOB contextDetermined by URL path (/biz/ → Business, /cib/ → CIB). No authored metadata needed.

9. Risks & Mitigations

▼
# ▲▼ Risk ▲▼ Impact ▲▼ Likelihood ▲▼ Mitigation ▲▼
R1Font hosting — WF Sans woff2 files may be large or require multiple weightsMediumLikelyAudit font variants. Host only Regular + Bold. Use font-display: swap. Subset if possible.
R2Former-legacy pages in new framework — /help/*, /about/governance/*, /privacy-security/* need content restructuringMediumConfirmedImport content into new-framework sections. Tabs remain. Sidebar becomes inline. Validate early in Steps 3–4.
R3Product-comparison block complexity — dense 4-column interactive gridHighConfirmedPurpose-built block with responsive collapse. Validate in Step 2.
R4Footnote/disclaimer system — superscript refs link to modal dialogs (~600+ pages)MediumConfirmedConvert to anchor-linked disclaimers block. Lose modal UX, preserve content.
R5Spanish mirror — ~387 pages must maintain parityMediumConfirmedSame infrastructure + /es/ prefix. Validate 5 pages before bulk.
R6Bot protection / CSP — aggressive Content-Security-PolicyLow–MedObservedMonitor import failures. Headless browser fallback.
R7Fragment authoring workflow — new pattern for authorsLow–MedLikelyDocument in authoring guide. Create example fragments in Step 11.
R8Page count discrepancy — sitemap ~1,067 vs. estimate ~1,300LowPossibleRe-crawl with broader discovery if gaps found.
R9Calculator stubs — users may expect working toolsMediumConfirmedClear "Visit our calculator" CTA. Phase 2 flag in metadata.
R10Single-project scale — ~1,067 pages in one DA projectLowPossibleDA handles this scale. Folder structure mirrors site IA.
