Two-line summary

Migrate ~1,067 pages from wellsfargo.com to Edge Delivery Services with Document Authoring (DA). The site is organized into 14 page templates. We will prefer default content (David's Model), then reuse the DA Block Collection, then Block Party aggressively, and build custom blocks only when no equivalent exists. Two palettes (Default + Wealth) drive sectioned theming.

Key Decisions (locked by user, v2)

D1Wealth theme. /investing-wealth/ uses a distinct color palette. Applied via section/page metadata theme: wealth. Default theme everywhere else, including Premier and CIB.
D2Block Party. Use Block Party blocks as much as possible. Reusability is the priority.
D3Header / Footer variants. Multiple header & footer variants confirmed (Default consumer, Biz, CIB / institutional, Wealth). Each variant authored as a separate /nav and /footer doc, selected by page metadata.
D4Spanish parity. Phase 1 ships the Spanish home page only. Other /es/ pages are deferred to a later phase.
D5Help framework. Help pages use the same theme/framework as the home page (no separate help framework).
D6Fonts. Download Wells Fargo Sans (and Serif if present) from the live site and self-host on EDS under /fonts/.
D7Image hosting. Migrate all images to DA content pages. No reliance on www17.wellsfargomedia.com.
D8Rates source. Manage rates in DA-hosted spreadsheets for now. Rates Table block reads from those sheets.
D9Calculators / tools. Phase 2 scope; Phase 1 stubs link to live URLs. (Strategy beyond Phase 1: open.)
D10Footnote / disclosure IDs. Strategy open; for Phase 1, author IDs verbatim per page (same as legacy).
D11Analytics. Out of scope for Phase 1. No tag manager wired.
D12Cookie consent. Copy live-site consent configuration as a fragment.
D13"How was your experience?" widget. Out of scope for Phase 1.
D14Sitemap / redirects ownership. Open. To be revisited before go-live.
D15External-domain content. No migration of external domains (creditcards, stories, oam, connect.secure, wellsoffice, www17 media). Links preserved.
D16Project structure. Single DA project, folders mirror site IA. /es/ subfolder for the Spanish home.
D17Placeholders & Configs. Use placeholders.json for phone numbers, sign-on URLs, app store links, FDIC strings. /configs/ for theme map, rates source, fragment paths.

What changed in v2 (vs. v1)

  • Adopted EMA's confirmed hex codes (#D71E28, #FCC60A, #5A469B, #F4F0ED, #3B3331, #787070, #468254) and concrete type sizes (17px body, 34px H1, 1400px max content width, 24px button pill).
  • Added two templates Claude missed: T13 Leadership Bio and T14 About Landing.
  • Split T8 Business into T8a Biz Landing + T8b Biz Sub-page per EMA's stronger carving.
  • Added Carousel and Table to the Block Collection adoption set; added resource-links and contact-info as custom blocks.
  • Doubled down on Block Party tier per D2: anything in Block Party is preferred over a new custom block.
  • Restored the Wealth palette (per D1) scoped to /investing-wealth/ only (not Premier or CIB — per user clarification).
  • Adopted EMA's effort-per-step convention (~28 days, padded from EMA's 23d to absorb Wealth theme + Block Party tier + image migration).
  • Added EN→ES sequencing (Phase 4 ES bulk after Phase 3 EN bulk; though now only ES home in Phase 1).
  • Out-of-scope: added test/dev pages and _navlinks/_reusable CMS fragments.
  • Risks: added product-comparison complexity, CSP/bot-protection, fragment authoring training, former-legacy restructuring.
  • Open Questions cut from 12 → 4 (D9 calc strategy beyond stubs, D10 footnote IDs, D14 sitemap ownership, plus a new question on header/footer mapping by page).

9. Risks & Mitigations

▶
High

Legal / disclosure drift

Banking disclosures (FDIC, NMLSR ID, footnote IDs like PM-…, DT2-…, LRC-1224) are tightly versioned and audited. Drift during bulk import = compliance risk.

Author every disclosure as a fragment with a single source of truth, dated. Pin the footnote IDs as content per D10. Final pre-publish diff against live for every page.
High

Image migration to DA at scale

Per D7, all marketing images (thousands across www17.wellsfargomedia.com) must be re-hosted in DA. Risk of broken refs and large asset uploads.

Dedicated tooling step (worklist step 10) with enumeration → download → upload → URL-rewrite map. Verify zero external image refs at QA (step 17). Cap concurrency to avoid DA upload throttling.
High

product-comparison block complexity

Dense 4-column interactive grid (checking comparison, savings comparison). Has to collapse responsively on mobile.

Purpose-built block with responsive table-to-stacked-card collapse below 768px. Validate in step 3 (T2 pilot). EMA risk R3 adopted.
High

Trademark / accessibility preservation

Live site has many ®, ™, skip links, SR-only spans that bulk scrapers can drop.

Import preserves trademark glyphs verbatim; skip-link and SR-only spans re-added at the template level, not per page.
Med

Font licensing

Wells Fargo Sans / Serif licensing may restrict self-hosting on a new EDS origin.

Per D6, fonts downloaded from live site and self-hosted. Confirm licensing legal review before step 4. Fallback: system stack already wired with font-display: swap.
Med

Calculator stubs going stale

Phase 1 points tools to live URLs (per D9). If live URLs change before Phase 2, links break.

Route tool links through placeholders ({{tool-url-personal-loan-calc}}) so swapping is a one-line config change at Phase 2.
Med

"How can we help?" strip variants

The strip varies by section (Call Us vs. Quick Help vs. Find a location).

Three named fragment variants: help-cta-default, help-cta-product, help-cta-biz. Page selects via metadata.
Med

Sitemap inflation / hidden URLs

Real authorable HTML count may differ from EMA's 1,067 baseline by ±15%.

Step-1 pre-flight: parse the sitemap, drop non-200 / non-HTML / auth/external. Re-baseline before bulk.
Med

CSP / bot protection on import

Live site has aggressive Content-Security-Policy and bot detection; bulk scraping may be throttled.

Monitor import failures. Headless browser fallback. Throttle requests to ~1/sec with retry/backoff. EMA risk R6 adopted.
Med

Fragment authoring workflow training

Fragments are a new pattern for authors (cookie-consent, app-promo, help-cta variants, disclosures).

Document the fragment index in the authoring guide (step 18). Create example fragments in step 7. EMA risk R7 adopted.
Med

Footnote modal preservation

Legacy footnotes use modal dialogs (~600 pages). Downgrading to anchor links loses UX.

Block Party Modal block preserves the modal pattern. Footnote refs in body become data-modal superscripts; bodies live in the Disclaimers block at page bottom.
Low

SEO regressions during import

Bulk imports can drop canonical, OG fields, structured data.

Preserve metadata block on every imported page; pre-publish metadata diff against live. Lighthouse SEO ≥ 95 gate per wave.
Low

Wealth theme bleed

Section-level theme could bleed across section boundaries.

CSS-variable swap scoped to the section wrapper, not the document. Default theme always inherited above and below.
Low

Single-project scale (~1,067 pages)

Authoring tool performance with one large DA project.

DA handles this scale. Folder structure mirrors IA. EMA risk R10 adopted.
