1. EDS Sitemap Checker

Say: "check sitemap for <url>"

Fetches the live site's sitemap, maps every URL to the EDS domain, and HEAD-checks HTTP status for each one. The foundation of any migration audit — run this first.

What it checks

StatusMeaningAction
200 OKPage is live on EDSNone needed
404Page not published / migratedPublish or create the page
REDIRECT301/302/307/308 redirectUpdate sitemap or review redirect rules
ERRORTimeout or network errorCheck EDS connectivity

Output files

FileContents
/tmp/eds-url-check.csvoriginal_url, eds_url, status, redirect_location
/tmp/eds-url-check.htmlInteractive report with LIVE / 404 / REDIRECT / ERROR badges, sorted worst-first
/tmp/sitemap-urls.jsonShared input file — reused automatically by all other skills

Options

FlagDescription
--auth=user:passHTTP Basic Auth for htaccess-protected environments

Performance

10 concurrent HEAD requests, 15s timeout per URL. ~73 URLs ≈ under 1 minute.

2. EDS Content Validator

Say: "run content audit" / "run content compare"

Three modes for different depths of content checking. Modes A & B compare production content against EDS. Mode C audits the EDS site itself — no production comparison required.

A — Fast Compare B — Deep Compare C — Site Audit ★ most common

Mode A — Fast Compare

HTTP fetch + Jaccard similarity scoring. No browser required.

Speed~2–3 min / 100 pages
Best forCI-style migration audits, quick parity check
ScoresMATCH (≥90%) / PARTIAL (50–90%) / MISMATCH (<50%)
ChecksFull-page text similarity, heading match, CTA match, word count delta

Mode B — Deep Compare

Playwright Chromium — real browser with full JavaScript execution.

Speed~8–12 min / 50 pages
Best forPages with accordions, tabs, carousels, lazy-loaded fragments
ExtraExpands accordions, scrolls to trigger lazy sections, strips cookie banners

Mode C — Site Audit

Audits the EDS site itself — no production comparison needed. Catches quality issues in authored content.

CategoryWhat it flags
COMPLETENESSLorem ipsum, placeholder/TODO text, [[templates]], ALL CAPS blocks, stub pages (<20 words)
IMAGESMissing alt text, broken image URLs (404)
VIDEOS<video> missing poster attribute
LINKSAbsolute links to prod domain, old CMS domain links, generic anchor text ("click here", "read more")
QUALITYNo H1 heading, multiple H1 headings
NAV/FOOTERNav and footer pages accessible, no broken links inside them

3. EDS SEO Validator

Say: "run seo audit" / "run deep seo audit" / "run CWV check"

Four modes covering everything from basic metadata to full Lighthouse scores and real-user CrUX field data.

A — Fast SEO B — Deep SEO ★ pre-launch C — CWV / PageSpeed D — SEO Compare

Mode A — Fast SEO

HTTP fetch + regex extraction. No browser required. ~2–3 min / 100 pages.

CheckRule
<title>Required · 30–60 chars
meta descriptionRequired · 120–160 chars
og:title / og:description / og:imageRequired · length-checked · og:image HEAD-checked for 200
og:url / og:typeRequired
twitter:cardRequired · valid values only
Duplicate titles/descriptionsFlagged across all pages

Mode B — Deep SEO

Full Playwright Chromium audit — 68 of 80 SEO checklist items. ~8–10 min / 73 pages.

SectionChecks
§1 MetadataTitle (50–60 chars), description (140–160 chars), no keywords tag, robots meta
§2 JSON-LDWebSite schema, BreadcrumbList on non-homepage, required fields per type, placement in <head>
§3 HeadingsSingle H1, no skipped heading levels (H1→H2→H3)
§4 CanonicalPresent, HTTPS, self-referencing
§5 Crawlabilityrobots.txt + sitemap referenced, custom 404
§6 URLLowercase, hyphens not underscores
§7 LinksGeneric anchor text, image links without alt, breadcrumb nav
§8 RedirectsHTTP→HTTPS (301 only), www consistency
§9 OG + TwitterAll required tags, og:image pixel dimensions (1200×630), twitter:site handle
§10 CWV (synthetic)TTFB ≤800ms, FCP ≤1800ms, LCP ≤2500ms, CLS ≤0.1 — via Playwright PerformanceObserver
§11 Images / A11yMissing alt attributes, buttons without accessible labels, favicon
§12 Mobilewidth=device-width viewport, no horizontal scroll at 390px
§13 Paginationrel=next/prev on paginated URLs
AggregateDuplicate titles, descriptions, H1 text across all pages

Generates both .csv and .html — the HTML report includes stat cards, colour-coded category badges, and PASS/FAILED/ERROR rows sorted worst-first.

Mode C — CWV / PageSpeed Insights

Uses Google PageSpeed Insights API for authoritative Lighthouse lab scores + real-user CrUX field data. URLs must be publicly accessible.

Data sourceMetrics
Lab (Lighthouse)Performance score (0–100), LCP, FCP, CLS, TTFB, TBT, Speed Index, TTI
Field (CrUX p75)LCP, FCP, CLS, INP (only measurable here), TTFB — rated GOOD / NEEDS_IMPROVEMENT / POOR

Default strategy is both — runs mobile then desktop sequentially and produces a single combined CSV + HTML with per-strategy stat cards.

Mode D — SEO Compare

Compares SEO metadata side-by-side between production and EDS — verifies that title, description, OG tags, and Twitter tags were migrated correctly (not just that they exist, but that values match).

4. EDS Visual Compare

Say: "run visual compare"

Full-page screenshot comparison between production and EDS across desktop, tablet, and mobile viewports. Uses Playwright + pixelmatch to diff pages pixel-by-pixel. Changed pixels highlighted in red.

Viewports

ViewportResolutionRepresents
Desktop1440×900Standard laptop/monitor
Tablet768×1024iPad portrait
Mobile390×844 (2×)iPhone 14

How it works

  1. Opens both prod and EDS in a real Chromium browser
  2. Scrolls the full page in 600px increments to trigger lazy-loaded images
  3. Takes full-page screenshots at each viewport
  4. Diffs with pixelmatch — changed pixels highlighted in red
  5. Flags pages where diff % exceeds the threshold (default 5%)

Status values

StatusMeaning
PASSDiff % ≤ threshold
FAILDiff % > threshold
PROD_BLOCKEDWAF blocked prod — EDS screenshot still saved
EDS_NOT_FOUNDEDS returned non-200 status

Output

FileContents
<output-dir>/index.htmlInteractive HTML: Prod | EDS | Diff side-by-side per page + viewport, sorted by largest diff first
<output-dir>/screenshots/page-NNNN/desktop/tablet/mobile prod, EDS, and diff PNGs

Options

FlagDefaultDescription
--threshold=N5% pixel diff to flag as FAIL
--max=NallLimit pages — use 5–20 for a quick first check
--viewports=...allComma-separated: desktop,tablet,mobile
--auth-prod=user:pass—Basic Auth for prod

Common Patterns

HTTP Basic Auth

All skills support htaccess-protected environments. The main agent probes the first URL before running — if it returns 401, credentials are requested in user:password format and passed as --auth=user:pass.

Note: CWV mode cannot use auth credentials — PageSpeed Insights crawls from Google's servers.

Shared sitemap file

Once eds-sitemap-checker runs and creates /tmp/sitemap-urls.json, all other skills reuse it automatically. No need to re-fetch the sitemap between audits in the same session.

Large sitemaps (500+ pages)

All skills support --max=N and --offset=N for batching. The main agent can split the sitemap into batch files and dispatch multiple parallel sub-agents simultaneously, then merge results.

Sub-agent pattern

All scripts are dispatched as background sub-agents (via the Agent tool) to keep the main conversation clean. Script output — which can be thousands of lines — is summarised and returned as structured tables.

Zip export

After any audit, you can ask to have the report saved as a zip in the project's testing/ folder for sharing or archiving.
